üìã Issue Description

Problem: Students added manually while offline were not being sent when the system reconnected.

Root Cause: The manual sync function only processed students from registeredByDate[today] but ignored the offlineRegistrations array.

üîß Fix Applied

The following changes were made to System/web-interface/scripts/Entry-Scanner.js:

1. Enhanced manualSync() function:

// Added offline registrations processing const offlineStudents = offlineRegistrations.filter(record => { const studentKey = `${record.student_id}_${record.student_name}_${new Date(record.timestamp).toDateString()}`; return !sentStudents.has(studentKey); }); const totalStudentsToSend = allStudents.length + offlineStudents.length; // Send both today's registrations AND offline registrations for (const student of offlineStudents) { // Send to server with duplicate prevention if (ws && ws.readyState === WebSocket.OPEN) { ws.send(JSON.stringify({ type: 'student_registered', record: student })); sentStudents.add(studentKey); totalSent++; } }

2. Enhanced showSyncDetails() function:

// Added offline registrations count const unsentOfflineRegistrations = offlineRegistrations.filter(record => { const studentKey = `${record.student_id}_${record.student_name}_${new Date(record.timestamp).toDateString()}`; return !sentStudents.has(studentKey); }).length; const totalPendingSync = unsentStudents + unsentOfflineRegistrations;

3. Enhanced forceSyncAllStudents() function:

// Added offline registrations to force sync const totalStudentsToSend = allStudents.length + offlineRegistrations.length; // Send offline registrations in force sync for (const student of offlineRegistrations) { if (ws && ws.readyState === WebSocket.OPEN) { ws.send(JSON.stringify({ type: 'student_registered', record: student })); totalSent++; } }
